# 阶段沟通总结 - AI 管培生入职介绍前端界面

更新时间：2026-06-10 14:40 CST

## 已确定的核心方案

1. 首页不是静态贴图，而是使用真实前端结构实现：顶部导航、健康元 Joincare 标识、主标题、按钮和代码雨背景都由 HTML、CSS、JavaScript 生成。
2. 主页面从左到右展示 12 张管培生个人生活照，照片下方按照“姓名在上、部门在下”的顺序展示，整体为圆弧形排列。
3. 鼠标滑过照片墙时，当前照片放大，两侧照片发生缩放和轻微位移，用来模拟 Mac Dock 的聚焦感。
4. 点击照片后进入抽卡详情态：左侧保留被点击者照片卡片，右侧铺开详情区，详情字段包括专业背景、AI 搭子、本命 AI 搭子、最想让 AI 解决的问题、AI 超能力、有趣事实。
5. 详情区下方保留电子盲盒入口；进入抽词云后由主持人手动输入句子，输入完成后句子替换原按钮。
6. 加载页使用健康元图标居中展示，背后为代码雨效果；加载动画节奏为“从暗到亮，再慢慢淡下来”。
7. 当前加载页停留时间为 4 秒，随后用 1.2 秒做加载页淡出和首页淡入的交叉过渡。

## 本阶段遇到的问题

1. 一开始主页面曾直接引用截图背景，这和 Jasper 要求的“严格按图复刻但不能贴图替代 UI”不一致，后来改成真实 DOM/CSS 实现。
2. 健康元图标文件虽然看起来像透明图，但实际 RGB 里带有白色或棋盘格背景，因此生成了清理后的透明资产用于前端展示。
3. 首页照片墙曾出现图片过小、圆弧叠压、顶部导航被遮挡等问题，后续通过卡片尺寸、圆弧布局、导航层级调整逐步修正。
4. 详情页信息字段曾遮住下方内容，后来改成一行一行展示，并让信息区可滚动。
5. 加载页切到首页曾显得生硬，根因是首页没有在加载页淡出前作为底层准备好；现在改为交叉过渡。
6. 自动化浏览器验证时页面处于后台状态，`document.visibilityState` 为 `hidden`。这会触发浏览器节流，所以采样时间不完全等同于用户实际可见窗口中的动画时间。

## 关键术语说明

- DOM：Document Object Model，中文可理解为“文档对象模型”。它是浏览器把 HTML 页面解析成的一棵结构树，按钮、图片、标题这些页面元素都在这棵树里。
- CSS：Cascading Style Sheets，中文是“层叠样式表”。它负责页面的颜色、尺寸、位置、过渡动画等视觉表现。
- JavaScript：浏览器里的脚本语言。当前项目用它控制页面状态切换、照片墙交互、详情弹窗和词云抽取。
- transition：CSS 过渡动画。它的逻辑是当某个样式值从 A 变成 B 时，浏览器在指定时间内平滑补间，而不是瞬间切换。
- crossfade：交叉淡入淡出。这里指加载页透明度从 1 降到 0，同时首页透明度从 0 升到 1。
- code rain：代码雨。这里是用 canvas 画布不断绘制字符流，形成科技感背景。
- canvas：HTML 画布元素。它不是普通图片，而是由 JavaScript 一帧一帧绘制出来的图形区域。
