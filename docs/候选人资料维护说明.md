# 候选人资料维护说明

## 核心文件

- `data/trainees.json`：候选人资料库。这里维护姓名、部门、个人背景、AI 字段、生活照路径、表情包路径。
- `src/data.js`：数据加载器。页面打开后会读取 `data/trainees.json`，如果读取失败，会使用 `src/app.js` 里的最小兜底数据。
- `src/logic.js`：数据标准化逻辑。`normalizeTrainee` 会把 `aiPartners` 映射成页面使用的 `tools`，把 `favoriteAI` 映射成 `favoriteTool`。
- `src/app.js`：页面渲染和交互。这里不建议手动改候选人资料，只负责读取数据并渲染界面。

## 新增或替换候选人

每个候选人在 `data/trainees.json` 里是一段 JSON 对象。建议保留以下字段：

```json
{
  "id": "xu-ran",
  "department": "战略部",
  "departmentEn": "STRATEGY",
  "name": "许然",
  "romanName": "Xu Ran",
  "background": "AI 产品策略 / 用户研究",
  "aiPartners": "ChatGPT / Gamma / 飞书多维表格",
  "favoriteAI": "ChatGPT",
  "aiProblem": "会议纪要自动归纳与跨部门追踪",
  "aiPower": "把复杂讨论压缩成一页清晰决策图",
  "funFact": "为了让汇报页对齐，曾经手动微调到凌晨。",
  "photo": "./assets/trainees/xu-ran/photo.png",
  "memeImage": "./assets/trainees/xu-ran/meme.png",
  "memeText": "OK, SHIP",
  "portrait": "linear-gradient(145deg, #051916 0%, #0f725f 48%, #d6fff0 100%)"
}
```

## 图片放置规则

推荐路径：

```text
assets/trainees/<候选人id>/photo.png
assets/trainees/<候选人id>/meme.png
```

例如 `id` 是 `song-lan`，则图片路径是：

```text
assets/trainees/song-lan/photo.png
assets/trainees/song-lan/meme.png
```

## 术语解释

- `id`：候选人的唯一标识，用来绑定卡片点击、图片路径和句子保存。
- `JSON`：一种结构化数据格式，适合维护候选人资料，不需要改 JS 代码。
- `fallback`：兜底数据。意思是外部 JSON 读取失败时，页面仍有最少数据可以展示。
- `portrait`：渐变占位背景。真实照片缺失时会显示它，避免页面出现破图图标。
