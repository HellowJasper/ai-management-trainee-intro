# 前后端改造说明

## 当前结构

项目现在仍保留原有静态页面，但默认运行方式已经切到 Node 服务：

- `server/index.js`：HTTP 服务入口，同时提供静态资源和 API。
- `server/traineeRepository.js`：候选人数据仓储层，当前读写 `data/trainees.json`。
- `src/data.js`：前端 API 客户端，优先读取 `/api/trainees`，失败后兜底读取 `data/trainees.json`。
- `src/app.js`：页面交互和渲染逻辑，主持人录入句子后会尝试保存到后端。

## 本地运行

```bash
npm start
```

默认地址：

```text
http://localhost:5173
```

如果端口被占用：

```bash
PORT=5174 npm start
```

仍可使用纯静态模式对比：

```bash
npm run static
```

## API

```text
GET    /api/health
GET    /api/trainees
POST   /api/trainees
GET    /api/trainees/:id
PATCH  /api/trainees/:id
DELETE /api/trainees/:id
PATCH  /api/trainees/:id/sentence
POST   /api/trainees/:id/sentence
```

目前 API 直接写回 `data/trainees.json`，适合本地和小范围内部演示。后续如果要多人同时维护、上传图片、权限管理，应把 `server/traineeRepository.js` 替换为数据库实现。

## 后续演进建议

1. 把 `src/app.js` 继续拆成视图模块，例如照片墙、详情弹层、盲盒挑战和路由状态。
2. 增加后台管理页，用 API 维护候选人资料和业务场景链接。
3. 增加图片上传接口，把图片保存到对象存储或服务器目录。
4. 引入数据库，例如 SQLite、PostgreSQL 或 MySQL。
5. 增加登录权限，区分展示页、主持人操作和管理员操作。
