# 角色登录与权限分流方案

依据飞书文档《2026 健康元药业 AI 星锐黑客松平台架构方案》中的「用户角色与权限」「移动端 · 选手 / 评委 / 大众同端分权」「四类用户可见页面汇总」「管理后台」「组队抢赛道」「评分与综合计算模型」「数据与权限模块」整理。

## 1. 核心结论

点击右上角「登录 / 角色」后，最终不应该让用户自由选择角色，而是通过飞书账号登录后，由后端根据账号返回角色身份。

角色分为四类：

| 角色 | 文档对应身份 | 页面定位 |
| --- | --- | --- |
| 管理员 | 会务 / 主持 / IT | 管理后台、流程控制、大屏控制、数据管理 |
| 参赛选手 | 管培生 / 内部员工 / 导师 | 抢赛道、组队、作品提交 |
| 专家评委 | 总裁 / CTO / 业务负责人 | 查看待评作品、五维评分、查看我的评分记录 |
| 大众评委 / 观众 | 普通员工 / 观众 | 浏览公开页面、在投票窗口内投票一次 |

需要严格遵守的权限边界：

- 只有参赛选手可以看到和操作「抢赛道 / 加入队伍 / 队伍工作台 / 作品提交」。
- 管理员、专家评委、大众评委都不能出现「加入队伍」「报名组队」「未加入队伍」这类选手操作文案。
- 组队进度是公开信息，四类角色都可以看，但非选手只能只读查看。
- 参赛选手与专家评委默认不参与大众投票。
- 大众评委只能在投票窗口内投一次票。
- 专家评委只能做专家评分，且结果发布前不能看到其他专家评分。
- 管理员不参与选手组队、投票、专家评分本身，而是管理流程、数据、权限、大屏与结果发布。

## 3. 登录 / 角色分流

### 3.1 当前前端临时逻辑

开发阶段可以保留一个模拟角色选择弹层，方便本地测试：

- 参赛选手
- 专家评委
- 大众评委
- 管理员

但这个只能作为 mock，不作为最终业务逻辑。

### 3.2 后端接入后的正式逻辑

点击「登录 / 角色」后：

1. 触发飞书登录。
2. 后端根据飞书账号识别用户。
3. 后端返回用户身份与权限。
4. 前端保存 session。
5. 根据角色进入对应角色工作台。

建议返回结构：

```json
{
  "userId": "feishu_open_id",
  "name": "张三",
  "avatar": "https://...",
  "role": "player",
  "permissions": [
    "team:join",
    "work:submit",
    "gallery:view"
  ],
  "teamId": "team_03"
}
```

角色枚举建议：

| role | 含义 |
| --- | --- |
| admin | 管理员 |
| player | 参赛选手 |
| judge | 专家评委 |
| public | 大众评委 / 观众 |

如果账号没有配置角色，不建议默认当作大众评委，应该显示「未分配角色，请联系管理员」。

## 4. 角色工作台展示规则

当前截图里的「角色工作台」需要按角色切换内容，不能所有人共用同一组卡片。

### 4.1 参赛选手

参赛选手可以展示：

- 当前身份：参赛选手
- 组队状态：未加入队伍 / 已加入队伍
- 赛道状态：可抢赛道 / 赛道已锁定
- 作品提交状态：未提交 / 已提交 / 审核中 / 已发布

快捷入口建议：

- 抢赛道 / 队伍工作台
- 作品提交
- 作品展厅
- 大赛流程

选手是唯一可以看到「加入队伍」「报名组队」「抢赛道」操作的角色。

### 4.2 专家评委

专家评委可以展示：

- 当前身份：专家评委
- 待评作品：未评分数量
- 评分进度：已评分 / 总作品数
- 我的评分记录：只看自己的提交记录
- 组队进度：只读查看

快捷入口建议：

- 待评作品
- 五维评分
- 我的评分记录
- 作品展厅
- 大赛流程

专家评委不应该看到：

- 未加入队伍
- 报名组队
- 加入队伍
- 大众投票入口

### 4.3 大众评委 / 观众

大众评委可以展示：

- 当前身份：大众评委
- 投票状态：未投票 / 已投票 / 投票未开放 / 投票已结束
- 组队进度：只读查看
- 作品展厅：可浏览

快捷入口建议：

- 投票入口
- 作品展厅
- 组队进度
- 路演进度 / 最终结果

大众评委不应该看到：

- 未加入队伍
- 报名组队
- 加入队伍
- 专家评分
- 我的评分记录

### 4.4 管理员

管理员可以展示：

- 当前身份：管理员
- 当前阶段：组队 / 开发 / 路演 / 投票 / 结果
- 组队管理：开关、微调、补全
- 投票管理：窗口、异常、数据
- 评分管理：进度、汇总
- 大屏状态：当前展示页、推送状态

快捷入口建议：

- 管理后台
- 流程控制
- 大屏控制
- 用户与权限
- 投票管理
- 评分管理
- 结果发布

管理员不应该看到：

- 未加入队伍
- 报名组队
- 加入队伍
- 选手作品提交入口
- 大众投票按钮

## 5. 当前界面里需要改掉的内容

### 5.1 「组队状态：未加入队伍」

这行只适合参赛选手。

其他角色应替换为：

| 当前角色 | 建议替换 |
| --- | --- |
| 专家评委 | 组队进度 / 仅可查看 / 查看各赛道名额与满员状态 |
| 大众评委 / 观众 | 组队进度 / 仅可查看 / 查看当前队伍形成情况 |
| 管理员 | 组队管理 / 可控场 / 开关组队、微调成员、补全空位 |

### 5.2 「快捷入口：报名组队」

这项只给参赛选手展示。

其他角色应替换为：

| 当前角色 | 替换入口 |
| --- | --- |
| 专家评委 | 待评作品 / 我的评分记录 |
| 大众评委 / 观众 | 投票入口 / 作品展厅 |
| 管理员 | 管理后台 / 大屏控制 |

### 5.3 「赛程进度：确认关键节点」

文档里没有要求把「关键节点」做成独立业务入口。这个更像是 AI 误加的功能文案。

建议改成更稳妥的公共入口：

- 大赛流程
- 路演进度
- 开发倒计时

如果后续文档没有明确「关键节点确认」这个操作，不建议继续保留「确认关键节点」。

## 6. 导航与快捷入口建议

### 6.1 顶部导航

公共导航建议保留：

- 首页
- 新生看板
- 赛程
- 作品展厅
- 最终排行

按角色追加：

| 角色 | 追加入口 |
| --- | --- |
| 参赛选手 | 组队 / 提交 |
| 专家评委 | 评委评分 |
| 大众评委 / 观众 | 投票 |
| 管理员 | 管理后台 / 大屏控制 |

### 6.2 移动端底部导航

移动端不适合把所有入口都塞进去，应按角色给 4-5 个高频入口。

| 角色 | 底部导航建议 |
| --- | --- |
| 参赛选手 | 首页 / 组队 / 提交 / 作品 / 我的 |
| 专家评委 | 首页 / 作品 / 评分 / 记录 / 我的 |
| 大众评委 / 观众 | 首页 / 作品 / 投票 / 结果 / 我的 |
| 管理员 | 控制台 / 大屏 / 数据 / 结果 / 权限 |

如果要弱化「个人中心」，可以把「我的」改成「角色」或「权限」。

## 7. 后端接口预留建议

后续接后端时，建议至少预留这些接口：

```txt
GET  /api/me
GET  /api/permissions
POST /api/auth/feishu/login
POST /api/auth/feishu/callback

GET  /api/team/progress
POST /api/team/join
GET  /api/team/my

GET  /api/work/my
POST /api/work/submit
GET  /api/gallery

GET  /api/vote/status
POST /api/vote/cast

GET  /api/judge/tasks
POST /api/judge/scores
GET  /api/judge/my-scores

GET  /api/admin/dashboard
POST /api/admin/phase
POST /api/admin/bigscreen/push
POST /api/admin/result/publish
```

前端权限判断建议抽成统一函数：

```js
const canJoinTeam = role === "player";
const canSubmitWork = role === "player";
const canVote = role === "public" && voteWindowOpen && !hasVoted;
const canScore = role === "judge";
const canAdmin = role === "admin";
```

所有按钮点击也要做二次校验，不能只靠隐藏 UI。

## 8. 验收标准

后续实现时可以按这几条判断是否正确：

- 大众评委登录后，不再出现「未加入队伍」「报名组队」「加入队伍」。
- 专家评委登录后，不再出现「未加入队伍」「报名组队」「投票入口」。
- 管理员登录后，不再出现选手态的「报名组队」和大众态的「投票」。
- 参赛选手登录后，可以看到抢赛道、队伍工作台、作品提交。
- 非参赛选手即使手动访问组队操作页，也不能执行加入队伍。
- 非大众评委即使手动访问投票页，也不能投票。
- 非专家评委即使手动访问评分页，也不能评分。
- 未配置角色的账号不自动降级为大众评委，而是提示联系管理员。
